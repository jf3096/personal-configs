# Compliance Reviewer

该技能用于“完成/修改”场景的最终质量与合规门禁。

## 核心职责

- 审查结论是否有真实证据支撑。
- 审查输出格式是否满足统一契约。
- 审查关键风险是否被识别并披露。
- 在证据不足或格式不合规时阻断结论。

## 何时使用

出现以下任一情况时必须使用：

1. 即将给出“完成/修改”结论（done/completed/fixed/implemented/modified）。
2. 已做代码或配置变更并准备向用户交付。
3. 任务包含风险判断（行为变化、架构影响、回归风险等）。
4. 无法验证，需要给出“未验证”说明。

## 统一输出契约（必须）

若回复包含“完成/修改”结论，最后一个非空行必须是以下之一：

- `状态：完成（已验证：<command/results>）`
- `状态：修改（已验证：<command/results>）`
- `状态：修改（未验证：<reason>）`

推荐默认使用：

- `状态：完成（已验证：<command/results>）`

## 证据标准（通用）

至少提供一种可观测证据：

- 测试命令与结果
- 运行命令与输出
- 日志/检查报告

并满足：

- 证据可追溯到具体命令或输出。
- 证据覆盖本次任务的核心意图。
- 关键边界或异常路径已验证，或明确说明未覆盖原因。
- 不用无证据推测表达（如 `should work`、`probably`、`theoretically`、`预期`、`理论上`、`应该可以`）。

## 通用审查流程（建议）

1. 明确结论类型：完成、修改、未验证修改。
2. 提取变更点与风险点。
3. 对照证据检查真实性与覆盖度。
4. 判断是否满足统一输出契约。
5. 给出通过/阻断结论，并输出标准状态行。

## 阻断条件（必须）

出现以下任一情况必须阻断“完成/修改”结论：

- 声称完成/修改但无可观测证据
- 使用推测性语言替代证据
- 缺失标准最终状态行
- 证据与任务意图明显不匹配

## Hook 用法

本技能附带检查脚本：`hooks/pre-completion-check.sh`

```bash
cat /tmp/response-draft.txt | bash hooks/pre-completion-check.sh
```

查看 Hook 是否触发及内部判定字段：

```bash
cat /tmp/response-draft.txt | COMPLIANCE_REVIEW_VERBOSE=true bash hooks/pre-completion-check.sh
```

退出码：

- `0`：通过
- `1`：阻断，需修改后重试

## 审查结论模板（可选）

```markdown
## 合规审查报告

### ✅ 通过项
- ...

### ⚠️ 风险项
- ...

### ❌ 阻断项
- ...

### 最终判定
- [ ] 批准
- [ ] 有条件批准
- [ ] 拒绝
```
