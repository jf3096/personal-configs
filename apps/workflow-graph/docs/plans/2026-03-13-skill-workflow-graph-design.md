# Skill 设计文档：基于树状决策图的 Workflow 可视化

## 1. 目标
设计一个可复用 skill：根据用户描述的 workflow / 流程 / 机制，输出可在 CLI 快速理解的可视化结果。

本阶段为设计，不包含实现代码。

## 2. 适用场景
- GitLab 仓库流程总结
- 代码结构与调用机制理解
- Skill 自身运行机制解释

## 3. 核心输出范式
统一采用：树状决策图（Tree Decision Diagram）。

输出形态固定为：
- 图本体（主路径 + 分支 + 循环引用）
- 每个节点一行解释（`::`）

循环表达统一格式：`[LOOP Lx -> (n)]`

## 4. 输出模式（中文）
支持两种模式：

1. 简版
- 仅保留：节点 + 一行解释
- 不输出节点子点
- 面向“快速扫读”

2. 详细版
- 保留：节点 + 一行解释
- 为关键节点提供子点
- 关键节点必须完整覆盖（不限制子点数量）
- 面向“机制拆解”

## 5. 模式选择策略
### 5.1 用户显式指定
- 用户说“简版” -> 使用简版
- 用户说“详细版” -> 使用详细版

### 5.2 用户未指定时
由 AI agent 自动判断，不强制追问。

自动判断建议规则：
- 任务复杂度低（主节点少、分支少、循环少） -> 简版
- 任务复杂度高（多分支、多循环、多对象协作） -> 详细版

建议阈值（可调）：
- `主节点 <= 7` 且 `分支数 <= 2` 且 `循环数 <= 1` -> 简版
- 否则 -> 详细版

### 5.3 用户体验补偿
即使 agent 自动选定模式，也在结果尾部提示可切换：
- “如需更细节可切换详细版”
- “如需更紧凑可切换简版”

## 6. 图结构规范
- 节点格式：`(n) 节点标题 :: 一行解释`
- 分支格式：`├─ Yes -> ...` / `└─ No -> ...`
- 回环格式：`[LOOP Lx -> (target)]`
- 编号全局唯一，便于引用和修订

## 7. 详细版子点规范
仅在详细版启用：
- 子点格式：`[n.m] 子点标题` + 简短说明
- 子点用于补充输入来源、约束、异常处理、验收标准等
- 子点数量不设上限，但应避免重复和空洞描述

## 8. 典型示例模板
```text
(1) 接收任务描述 :: 汇总输入并结构化
└─ (2) 信息是否完整? :: 校验最小生成条件
   ├─ Yes -> (3) 生成初版图 :: 输出树状决策图
   │          └─ (4) 用户是否看懂? :: 可读性验收门
   │             ├─ Yes -> (5) 输出最终图 :: 结束
   │             └─ No  -> (6) 调整标注与层级 :: 优化可读性 -> [LOOP L2 -> (3)]
   └─ No  -> (7) 追问缺失信息 :: 补齐关键缺口 -> [LOOP L1 -> (1)]
```

## 9. 非目标（当前阶段不做）
- 不绑定具体渲染引擎（Mermaid/PlantUML/DOT）
- 不实现自动抓取或代码解析器
- 不在本阶段定义执行器/命令行参数细节

## 10. 验收标准
- 用户可在 10~20 秒内读出主路径
- 可定位所有关键分支与循环
- 简版与详细版可稳定切换
- 用户未指定模式时，agent 自动决策可解释
