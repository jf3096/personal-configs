# Workflow Graph Skill

## 目标
将 workflow / 流程 / 机制描述转换为可在 CLI 快速阅读的树状决策图。

## 适用场景
- GitLab 仓库流程总结
- 代码结构与调用机制解释
- Skill 运行机制说明

## 输出契约
输出必须是“图本体优先”，节点格式统一为：
- `(n) 节点标题 :: 一行解释`

分支与循环格式：
- 分支：`├─ Yes -> ...` / `└─ No -> ...`
- 循环：`[LOOP Lx -> (n)]`

## 模式
1. 简版
- 仅输出节点与一行解释
- 不输出子点

2. 详细版
- 输出节点与一行解释
- 覆盖所有关键节点
- 可包含任意数量子点（例如 `[1.1] 输入来源`）

## 模式选择策略
1. 用户显式说“简版”或“详细版”：严格遵循用户指定。
2. 用户未声明时：根据复杂度自动判断。
- 复杂度低（主节点少、分支少、循环少）：默认简版。
- 复杂度高（多分支、多循环、多对象协作）：默认详细版。

## 响应规则
- 默认只输出图，不追加总结性列表。
- 若用户明确要求解释，再追加解释文本。
- 当自动选择模式时，可在图后用一句提示可切换模式。
